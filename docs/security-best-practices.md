# Security Best Practices

This document outlines the security practices that can improve the monitoring, detection, and overall security of the SOC lab.

---

# Keep Systems Updated

Regularly update the operating systems and installed applications to reduce the risk of known vulnerabilities.

---

# Use Strong Authentication

- Use strong passwords for all systems.
- Avoid sharing administrator credentials.
- Change default credentials before deploying services.

---

# Monitor Critical Files

Enable File Integrity Monitoring (FIM) for important system files and directories to detect unauthorized changes.

Examples include:

- Windows Registry
- `/etc/passwd`
- `/etc/shadow`
- `/etc/ssh/sshd_config`

---

# Review Alerts Regularly

Frequently review alerts generated by Wazuh to identify suspicious activities and verify whether they require further investigation.

---

# Secure Remote Access

- Allow SSH access only to authorized users.
- Disable unused services.
- Regularly review SSH configuration.

---

# Monitor Privileged Activities

Administrative actions should be monitored to identify unauthorized privilege escalation attempts.

Examples include:

- `sudo` usage
- User creation
- User deletion

---

# Protect Log Files

Logs should be protected against unauthorized modification or deletion because they are essential during incident investigations.

---

# Validate Configuration Changes

Any changes to Wazuh, Auditd, Sysmon, or system configuration files should be verified after implementation to ensure monitoring continues to work as expected.

---

# Perform Regular Testing

Generate test events periodically to verify that:

- Agents are connected.
- Logs are collected successfully.
- Detection rules are functioning correctly.
- Alerts are visible in the Wazuh Dashboard.

---

# Conclusion

Following these practices helps improve the reliability of the SOC lab and ensures that security events are collected, monitored, and investigated effectively.
